# Security Policy
This document provides guidelines for reporting security vulnerabilities in starrocks connector for flink.

## What should and should NOT be included?
Before reporting a vulnerability, please first review the security models of both StarRocks and Flink. Some issues that may appear to be vulnerabilities could be expected behaviors or out of scope for the starrocks connector for flink, and therefore not valid here.

Since this connector serves as a plugin for StarRocks to load data from Flink, vulnerabilities inherent to Flink itself are generally not addressed in this project. Instead, they should follow the Flink community’s official resolutions and release cycles.

## How to report a vulnerability?
To ensure responsible disclosure, please avoid submitting security issues via public GitHub issues or pull requests. Instead, please report potential vulnerabilities by sending the details to [support@starrocks.com](mailto:support@starrocks.com).
